# Changelog

All notable changes to this project will be documented in this file.

## [v2.0.0] — 2026-07-06

### 🎨 新增：12 种视觉风格库
- 新增 `references/style-library.md`，定义 12 种小红书封面视觉风格
- 每次风格选择时，AI 根据内容推荐 3 个最合适的风格，同时展示完整 12 种
- 支持按视觉形式选（编号）或按内容用途选（"课程教程""产品测评"等）
- 支持"延续风格"和"换个风格"快捷操作

### 📊 新增：A/B 测试机制
- 当用户在单选步骤选了 2 个选项时，询问输出方式：
  - 两套完整提示词（A/B 两张封面完整输出）
  - 一套主方案 + 一个变体（快速测试）
- 当用户选 3 个以上时，引导收敛到 2 个
- 新增 A/B 测试建议输出（点击/转化对比、观察指标建议）

### ✍️ 新增：封面文字与发布标题分工
- 新增 `references/cover-methodology.md`，定义封面方法论
- 封面文字（3-8 字）：负责点击触发，用矛盾、反常识、损失、数字、悬念
- 发布标题（15-24 字）：负责搜索入口和内容解释
- 两者互补，不再是简单拆分关系

### 🔄 增强：连续生成模式
- 业务画像支持快捷沿用："沿用上一次业务画像"
- 用户只需补充本次内容主题，不用每次都重新填写身份信息
- "全部重新开始"不等于清空历史，仍保留业务画像和风格用于后续沿用

### 👤 增强：人物保真
- 提示词中加入更严格的人物保真约束
- 为豆包等容易改脸的平台，提供专用锁脸提示词模板
- 明确：必要时可 100% 保留原图人物，只替换背景、标题和辅助元素

### 📁 文件结构更新
- 新增 `references/style-library.md`（12 种风格库）
- 新增 `references/cover-methodology.md`（封面方法论）
- 更新 `SKILL.md`：纳入风格选择、A/B 测试、标题分工、业务画像沿用
- 更新 `rick-通用版-Claude豆包KimiDeepSeek-小红书封面助手系统提示词.md`：同步所有新机制
- 更新 `README.md`：全面反映 v2.0 功能
- 新增 `CHANGELOG.md`（本文件）
- 新增 `rick-GitHub发布打包说明.md` 和 `rick-Kimi上传GitHub执行说明.md`

## [v1.0.0] — 2025-07

### 初始版本
- Codex Skill 版（`$rick-xhs-cover-strategist`）
- Claude / 豆包 / Kimi / DeepSeek 通用系统提示词版
- 三轮对话收集：形象图 → 业务画像 → 配套素材
- 5 种封面重点：强痛点 / 强结果 / 强产品 / 强人设 / 强对比
- 连续生成模式：同一对话内复用规范做多张封面
- 人物保真：保持原图人物身份，不重绘换脸
- 隐私脱敏：所有示例使用虚构人物和虚构品牌
