## [v1.1.0] — 2026-07-06

### 🎨 新增：12 种视觉风格库
- **v1.0 没有**：用户只能凭感觉选封面风格，封面容易千篇一律
- **v1.1 新增**：每次展示完整 12 种风格，AI 根据内容推荐 3 个最合适的
- 支持按视觉形式选（编号）或按内容用途选（"课程教程""产品测评""案例结果""个人IP"）
- 支持"延续风格"和"换个风格"快捷操作
- 新增文件：`references/style-library.md`

### 📊 新增：A/B 测试机制
- **v1.0 没有**：用户一次只能生成一张封面，无法对比不同方向
- **v1.1 新增**：当用户同时选 2 个方向时，提供两种输出模式：
  1. **两套完整提示词**：A/B 两张封面都完整输出，适合真正测试两种方向
  2. **一套主方案 + 一个变体**：主方案完整输出，变体只给差异化提示词，适合快速测试
- 选 3 个以上时引导收敛到 2 个
- 新增 A/B 测试建议输出（点击/转化对比、观察指标建议）

### ✍️ 新增：封面文字与发布标题分工
- **v1.0**：封面文字和发布标题简单拆分，两者功能模糊，用户容易混淆
- **v1.1 新增**：
  - **封面文字（3-8 字）**：负责点击触发，用矛盾、反常识、损失、数字、悬念
  - **发布标题（15-24 字）**：负责搜索入口和内容解释，含关键词、数字、结果、情感词
  - 两者互补，不再是简单拆分关系
- 新增文件：`references/cover-methodology.md`

### 🔄 增强：连续生成模式
- **v1.0**：业务画像复用需要每次手动说明，不够丝滑
- **v1.1 增强**：
  - 新增"沿用上一次业务画像"快捷选项
  - 用户只需补充本次内容主题，不用每次都重新填写身份信息
  - "全部重新开始"不等于清空历史，仍保留业务画像和风格用于后续沿用
  - 三张选项卡："1. 沿用人设和封面规范" / "2. 沿用人设但重新输入业务画像" / "3. 全部重新开始"

### 👤 增强：人物保真
- **v1.0**：基础保真约束，豆包等工具容易改脸
- **v1.1 增强**：
  - 提示词中加入更严格的人物保真约束（身份、年龄、肤色、姿态、发型、五官、气质等）
  - 为豆包等容易改脸的平台，提供专用锁脸提示词模板
  - 明确：必要时可 100% 保留原图人物，只替换背景、标题和辅助元素
  - 连续生成模式下默认：保持同一人设 + 面部特征

### 📁 文件结构更新
- 新增 `references/style-library.md`（12 种风格库）
- 新增 `references/cover-methodology.md`（封面方法论：标题逻辑与点击钩子）
- 更新 `SKILL.md`：纳入风格选择、A/B 测试、标题分工、业务画像沿用
- 更新 `rick-通用版-Claude豆包KimiDeepSeek-小红书封面助手系统提示词.md`：同步所有新机制
- 更新 `README.md`：全面反映 v1.1 功能，新增版本对比表
- 新增 `CHANGELOG.md`（本文件）
- 新增 `rick-GitHub发布打包说明.md` 和 `rick-Kimi上传GitHub执行说明.md`

## [v1.0.0] — 2025-07

### 初始版本
- Codex Skill 版（`$rick-xhs-cover-strategist`）
- Claude / 豆包 / Kimi / DeepSeek 通用系统提示词版
- 三轮对话收集：形象图 → 业务画像 → 配套素材
- 5 种封面重点：强痛点 / 强结果 / 强产品 / 强人设 / 强对比
- 连续生成模式：同一对话内复用规范做多张封面
- 人物保真：保持原图人物身份，不重绘换脸
- 隐私脱敏：所有示例使用虚构人物和虚构品牌
