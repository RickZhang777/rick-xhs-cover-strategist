## 这是什么？

**Rick XHS Cover Strategist** 是一个小红书封面制作的**对话式提示词工作流**。

它不是给你一段固定 prompt 让你自己改，而是像一位封面设计师一样，先问你几个问题，了解你的形象、业务和素材，然后帮你：

1. **判断封面策略** — 这张封面重点推什么（痛点 / 结果 / 产品 / 人设 / 对比）
2. **选择视觉风格** — 从 12 种风格库中推荐最适合你的 3 个，你选 1 个
3. **生成标题方案** — 封面文字（负责点击）和发布标题（负责搜索）分开设计，各给 3-5 个选项
4. **输出生图提示词** — 一段可以直接复制到 GPT / 豆包 / 其他生图工具里的完整提示词
5. **出图复核** — 提供可打印的 checklist，确保封面在信息流里能抓住眼球
6. **沉淀封面规范** — 做完第一张后，后续封面可以复用规范，不用每次都重新介绍自己

**一句话：你提供照片和业务信息，它帮你产出「能直接喂给 AI 画图工具」的封面提示词。**

## 完整工作流演示

### 第一次做封面（三轮对话）

**第 1 轮：上传形象图**
```
请上传至多 5 张个人形象图。
尽量选择脸部轮廓清晰、能识别出人物的照片。
```

**第 2 轮：描述业务画像**
```
- 你是谁：
- 服务谁：
- 解决什么问题：
- 这篇内容讲什么：
- 希望用户看完做什么：
```

如果上一次已有业务画像，系统会直接提供：
```
沿用上一次业务画像
```
你只需要补充本次内容主题即可。

**第 3 轮：上传配套素材**
```
- 产品原型 / App截图 / 网页截图
- Logo / 工具界面 / 产品图
- 课程页 / 交付物截图 / 案例截图
- 数据结果 / 用户评价 / 证明材料
- 过往封面 / 对标账号封面 / 品牌色参考
```

然后选择封面重点 → 选择视觉风格 → 选择标题 → 拿到最终提示词。

---

### 再做一张（复用模式，不用重新走三轮）

同一次对话里，做完第一张后，你可以直接说：

```
沿用上次规范，再做一张，主题是……
```

或者：
```
下一张
换个主题
继续生成
延续风格
换个风格
```

助手会自动复用你的人设、形象图和封面规范，只问新主题和新素材。你也可以选择：

```
1. 沿用上次人设和封面规范，只换本次主题/素材
2. 沿用人设，但重新输入业务画像
3. 全部重新开始
```

---

### 风格选择（12 种视觉风格）

每次展示完整 12 种风格，AI 根据你的内容推荐 3 个最合适的：

```
1. 爆款大字压顶风
2. 巨字拆分冲击风
3. 痛点红标警示风
4. 商业顾问杂志风
5. 访谈人物海报风
6. 白板框架讲解风
7. 产品界面主视觉风
8. 工具流程信息图风
9. 深色效率工作流风
10. 教程清单风
11. 小白科普问答风
12. 案例结果证明风
```

选择方式：
- 按视觉形式选：回复编号，例如 4
- 按内容用途选：回复"课程教程""产品测评""案例结果""个人IP"
- 想延续上一张：回复"延续风格"
- 想换掉上一张：回复"换个风格"

---

### A/B 测试

如果你同时选了 2 个方向，助手会问：

```
你希望我怎么输出？
1. 生成两套完整提示词：A/B 两张封面都完整输出，适合真正测试两种方向
2. 生成一套主方案 + 一个变体：主方案完整输出，另一个只给差异化提示词，适合快速测试
```

## 核心特点

| 特点 | 说明 |
|:---|:---|
| 🎯 **策略优先** | 不是直接给 prompt，而是先判断「这张封面要推什么」 |
| 🎨 **12 种风格库** | 从 12 种视觉风格中推荐，避免封面千篇一律 |
| 🔄 **连续生成** | 同一对话内可以连续做多张封面，复用规范和素材 |
| 📊 **A/B 测试** | 支持两套完整方案对比，科学测试封面效果 |
| ✍️ **标题分工** | 封面文字（负责点击）和发布标题（负责搜索）分开设计 |
| 👤 **人物保真** | 强调保持原图人物身份，不重绘、不换脸、不美化成陌生人 |
| 🛡️ **隐私安全** | 所有示例使用虚构身份，不涉及作者真实信息 |
| 🤖 **多平台适配** | Codex / Claude / 豆包 / Kimi / DeepSeek 都能用 |

## 生图工具建议

| 工具 | 推荐程度 | 说明 |
|:---|:---|:---|
| **GPT / Codex 生图** | ⭐⭐⭐ 首选 | 人物相似度最高，封面完成度最好 |
| **豆包** | ⭐⭐ 可用 | 需使用更保守的提示词，强制锁定人物原貌 |
| **Kimi** | ⭐ 不推荐生图 | 更适合做策略、标题、提示词整理；生图质感不稳定 |
| **Claude / DeepSeek** | 仅生成提示词 | 这些模型本身不支持生图，只负责出 prompt |
