## 二、项目定位

`rick-xhs-cover-strategist` 是一个对话式小红书封面策略与生图提示词生成工具。

它不是单一 prompt 模板，而是一套轻量工作流：

- 收集个人形象图
- 收集或沿用业务画像
- 收集产品、课程、案例、截图等配套素材
- 判断封面重点
- 选择封面视觉风格
- 生成封面文字和发布标题
- 生成可直接喂给 AI 生图工具的提示词
- 沉淀下一张封面可复用的规范
- 支持同一聊天里连续生成多张封面

适合人群：

- 小红书博主
- 知识 IP
- 课程、咨询、服务型账号
- 产品、工具、SaaS、AI 工具测评账号
- 需要批量制作小红书封面提示词的内容创作者

## 五、最近沟通后已经确定的核心规则

### 1. 连续生成模式

这个工具不是“一张封面一个新对话”。

用户可以在同一个聊天里连续生成很多张封面。

完成第一张封面后，助手需要记住：

- 上一张业务画像
- 上一张封面风格
- 上一张封面重点
- 上一张封面文字逻辑
- 上一张素材使用逻辑
- 个人形象图使用方式

当用户继续做下一张时，助手先问：

```text
这次封面要怎么沿用上次信息？回复编号即可：
1. 沿用上次人设和封面规范，只换本次主题/素材
2. 沿用人设，但重新输入业务画像
3. 全部重新开始
```

关键补充：

`全部重新开始` 不等于清空历史。

它只代表当前封面重新收集照片、素材和主题，但仍然保留上一张业务画像和上一张风格。

后续仍要提供：

```text
沿用上一次业务画像
沿用上一次风格
```

这里的“上一次”永远指最近完成的一张：

- 第 5 次生图，沿用第 4 次
- 第 12 次生图，沿用第 11 次

### 2. 业务画像可沿用

因为很多用户的服务客户和账号定位不会每天变化，所以重新做图时，不应该每次都重新问：

- 你是谁
- 服务谁
- 解决什么问题
- 希望用户看完做什么

当已有历史业务画像时，助手应该提供快捷选项：

```text
沿用上一次业务画像
```

用户选择后，只需要补充这一次内容主题：

```text
- 这篇内容讲什么：
- 有没有新的产品/案例/截图素材：
```

### 3. 风格限定

之前的问题是生成出来的封面容易变成同一种类型。

现在必须在“封面重点”之后加入“风格选择”。

规则：

- 每次展示完整 12 个风格。
- AI 根据当前内容推荐 3 个最合适的风格。
- 用户可以按视觉形式选编号。
- 用户也可以按内容用途回复，例如“课程教程”“产品测评”“案例结果”“个人IP”。
- 如果用户说“延续风格”，就沿用最近一张封面的风格。
- 如果用户说“换个风格”，就推荐 3 个和上一张明显不同的风格。

完整 12 个风格：

```text
1. 爆款大字压顶风
2. 巨字拆分冲击风
3. 痛点红标警示风
4. 商业顾问杂志风
5. 访谈人物海报风
6. 白板框架讲解风
7. 产品界面主视觉风
8. 工具流程信息图风
9. 深色效率工作流风
10. 教程清单风
11. 小白科普问答风
12. 案例结果证明风
```

详细说明在：

```text
references/style-library.md
```

### 4. 多选和 A/B 测试

如果某一步本来让用户单选，但用户选择了多个选项，不要直接判错。

规则：

- 用户选 2 个：先问用户是要“两套完整提示词”，还是“一套主方案 + 一个变体”。
- 用户选 3 个以上：提醒用户先收敛到最想测试的 2 个。
- 用户明确说“都生成”“A/B测试”“都要完整做”：直接生成两套完整方案。
- 如果只是标题多选，且策略和风格相同，优先建议“主方案 + 变体”。

询问模板：

```text
收到，你选了 X 和 Y。这两个方向都可以测。

你希望我怎么输出？回复编号即可：
1. 生成两套完整提示词：A/B 两张封面都完整输出，适合真正测试两种方向
2. 生成一套主方案 + 一个变体：主方案完整输出，另一个只给差异化提示词，适合快速测试
```

如果用户选 3 个以上：

```text
你选了 3 个以上方向，我建议先测试其中 2 个，否则输出会太散。

请保留最想测试的 2 个编号即可，例如：1 和 5。
```

### 5. 封面文字和发布标题分工

封面文字不要只是发布标题的拆分版。

分工：

- 封面文字：用于图片大字，负责点击触发，优先 3-8 个字或 2 行短句。
- 发布标题：用于小红书笔记标题，负责解释内容方向和搜索入口，约 15-24 字。

封面文字优先使用这些钩子：

```text
A 不等于 B
越 A 越 B
做了 X 却 Y
你以为 A，其实 B
只有一个关键
就这一件
真相 / 潜规则 / 没人说
```

商业类内容的人群词要尽量前置，例如：

```text
老板 / 生意 / 获客 / 成交 / 合伙人 / 外包 / 客服 / 团队
```

详细说明在：

```text
references/cover-methodology.md
```

### 6. 人物保真

人物保真是高优先级规则。

提示词中必须强调：

```text
人物必须最大程度贴近原图本人：脸型、五官比例、眼镜、发型、年龄感、气质、身材比例和服装都要接近参考图。

不要把人物美化、重绘、换脸或改成另一个人。

如果生图模型容易改脸，必要时可以 100% 保留原图人物形象，只重新设计背景、标题和辅助元素。
```

豆包或其他容易改脸的平台，需要额外强调：

```text
人物必须沿用参考图原始人物，不要重新生成另一张脸；保持原图脸型、五官、眼镜、发型、年龄感和真实气质，必要时直接保留原图人物，只替换背景、标题和卡片元素。

构图保持简单：一个主标题、一个人物、一个证明卡片、最多一个辅助小标题。不要复杂装饰，不要多层半透明效果，不要大量小字标签。人物完整清晰，不裁掉半张脸，不把脸融进背景。
```

## 六、核心工作流

第一次完整生成时：

```text
1. 上传个人形象图
2. 描述或沿用业务画像
3. 上传配套素材
4. 选择封面重点
5. 选择视觉风格
6. 选择封面文字 / 标题
7. 输出最终提示词
8. 沉淀本次封面规范
```

输出结构：

```text
1. 封面策略判断
2. 标题方案
3. 视觉方案
4. 小红书封面提示词
5. 出图复核清单
6. 本次封面规范沉淀
```

如果进入 A/B 测试：

```text
A 方案
- 封面策略判断
- 标题方案
- 视觉方案
- 小红书封面提示词
- 出图复核清单

B 方案
- 封面策略判断
- 标题方案
- 视觉方案
- 小红书封面提示词
- 出图复核清单

A/B 测试建议
- 哪个更适合点击
- 哪个更适合信任/转化
- 建议观察什么指标或用户反馈
```

## 七、平台建议

优先推荐用于最终生图。

优点：

- 人物相似度更稳定
- 对复杂提示词理解较好

可用，但提示词要更保守。

注意：

- 容易改人物脸
- 构图不要太复杂
- 要强锁人物原貌

不建议作为最终生图主力。

更适合：

- 整理信息
- 生成标题
- 梳理策略
- 写提示词
- 复盘图片问题
- 上传和维护 GitHub 仓库

### Claude / DeepSeek / Hermes

适合文本侧工作：

- 对话收集
- 标题策略
- 提示词生成
- 工作流执行

如果平台本身不支持生图，就只负责输出提示词。
