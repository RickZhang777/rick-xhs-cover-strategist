# Session Cover Standard

Use this only after a cover is completed or when the user asks to reuse previous style.

Keep the standard short. It is a working memory for the current chat, not a full brand manual.

## What to Capture

- Person usage: main photo type, face size, expression direction, and whether person is central or side-positioned.
- Title style: large title tone, hook type, subtitle pattern, and whether it uses pain/result/product/person emphasis.
- Visual tone: professional, sharp, friendly, founder-like, tool-like, premium, or high-click.
- Material usage: product screenshot, logo, prototype, case proof, dashboard, course material, or no material.
- Color logic: high contrast, dark/clean background, fixed title color if provided, or current best-performing color relation.

## What Not to Capture

- Exact coordinates or pixel placement.
- Too many color values.
- Long brand explanations.
- One-off details that only fit the current topic.

## Reuse Behavior

If the user says to reuse the previous standard, keep the above stable and only redesign:

- current topic
- title
- supporting material use
- cover emphasis
- prompt details

If the new topic conflicts with the previous standard, say so briefly and propose one adjustment.
